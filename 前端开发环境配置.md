# 前端开发环境配置

## 1. 必需软件

### Node.js
- 版本：v20.x 或更高（当前环境：v24.11.0）
- 下载：https://nodejs.org/
- 验证：`node -v` 和 `npm -v`

### pnpm（包管理器）
Vben Admin 使用 pnpm 管理依赖，必须安装：
```bash
npm install -g pnpm
```

### Git
- 下载：https://git-scm.com/

---

## 2. 克隆项目

```bash
git clone https://github.com/Lin-0408-Yiran/structural-design-system.git
cd structural-design-system
git checkout dev
```

---

## 3. 前端框架说明

前端采用 **Vben Admin v5.7.0**（`@vben/web-antd`），这是一个基于 Vue 3 的企业级后台管理框架。

项目结构：
```
frontend/                        # Vben Admin monorepo 根目录
├── apps/
│   └── web-antd/                # 实际应用（Ant Design Vue 版本）
│       ├── src/
│       │   ├── api/             # API 接口（auth、design、admin）
│       │   ├── components/
│       │   │   ├── design/      # 设计相关组件（ResultViewer、ProgressTracker、AskHumanModal）
│       │   │   └── visualization/ # 可视化组件（MarkdownViewer、DXFViewer、ChartViewer）
│       │   ├── views/
│       │   │   ├── structural/  # 结构设计页面（列表、新建、详情）
│       │   │   └── admin/       # 管理页面（用户管理、角色管理）
│       │   └── router/
│       │       └── routes/modules/
│       │           └── structural.ts  # 路由配置
│       ├── .env.development     # 开发环境配置
│       └── vite.config.ts       # Vite 配置（API代理）
└── packages/                    # Vben Admin 内部包
```

---

## 4. 安装依赖

```bash
cd frontend

# 批准构建脚本（首次安装后执行）
pnpm approve-builds
# 选择全部（按 a 键），然后回车确认

pnpm install
```

---

## 5. 启动前端开发服务器

```bash
cd frontend
pnpm dev
```

出现选择菜单时，选择 **`@vben/web-antd`**，然后回车。

访问：**http://localhost:5666**

---

## 6. 代理配置

`apps/web-antd/vite.config.ts` 已配置代理，无需修改：
- `/api/*` → `http://localhost:8000`（FastAPI 后端）
- `/ws/*` → `http://localhost:8000`（WebSocket）

---

## 7. 环境变量

`apps/web-antd/.env.development`：
```
VITE_PORT=5666
VITE_GLOB_API_URL=/api
VITE_NITRO_MOCK=false
```

## 10. 常见问题

**Q：启动后提示 `pnpm approve-builds`**
```bash
pnpm approve-builds
# 按 a 全选，回车确认
```

**Q：登录后看不到"系统管理"菜单**
- 确认使用管理员账号登录
- 在浏览器控制台执行 `localStorage.clear()` 后刷新重新登录

**Q：新建设计提示失败**
- 确认 Redis 已启动：双击项目根目录的 `启动Redis.bat`
- 确认 Celery Worker 已启动（见后端配置文档）
