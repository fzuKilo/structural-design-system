### 🟠 第二批：API Key和配额管理（3-4天）

#### 4. API Key加密存储（后端）
- **位置**: `backend/api/routes/auth.py`
- **技术**: cryptography库（AES加密）
- **功能**:
  - 加密服务：`encrypt_api_key()` / `decrypt_api_key()`
  - 更新接口：保存时加密，读取时解密
  - 环境变量：加密密钥存储在环境变量中

#### 5. 用户配额管理系统（后端）
- **配额检查中间件**:
  - 创建任务前检查配额
  - 配额不足时返回403错误
- **配额扣减逻辑**:
  - 任务完成后扣减配额
  - 记录配额使用历史
- **定时重置**:
  - Celery Beat定时任务
  - 每日0点重置daily配额
  - 每月1号重置monthly配额

#### 6. 前端API Key管理页面
- **位置**: `frontend/apps/web-antd/src/views/_core/profile/`
- **功能**:
  - 新增"API Key设置"标签页
  - 输入框（支持显示/隐藏密码）
  - 保存按钮 + 验证提示
  - 加密传输（HTTPS）

#### 7. 前端配额显示功能
- **仪表盘**:
  - 显示今日剩余配额
  - 显示本月剩余配额
  - 配额使用进度条
- **个人中心**:
  - 配额详情（每日/每月限额）
  - 配额使用历史图表
- **提示**:
  - 配额不足时的警告提示
  - 配额耗尽时的错误提示

### 🟡 第三批：进度反馈优化（2-3天）

#### 8. 设计进度条优化
- **多方案生成时**:
  - 显示"正在生成第X/3个方案"
  - 每个方案的子进度（设计中/分析中/评估中）
- **code_check时**:
  - 显示"正在校核规范条文"
  - 显示校核进度（已检查X/总共Y项）
  - 实时显示发现的违规项
- **实现方式**:
  - 后端通过WebSocket推送细粒度进度
  - 前端进度条组件支持嵌套进度

#### 9. Token自动刷新机制
- **位置**: 前端HTTP拦截器
- **逻辑**:
  - 检查Token过期时间
  - 提前5分钟自动刷新
  - 刷新失败时跳转登录页
- **后端接口**: `/api/auth/refresh`（需新增）

#### 10. WebSocket断线重连优化
- **位置**: 前端WebSocket管理器
- **策略**: 指数退避
  - 第1次: 1秒
  - 第2次: 2秒
  - 第3次: 4秒
  - ...
  - 最大: 60秒
- **UI提示**:
  - 连接中断时显示"正在重连..."
  - 重连成功后自动恢复
  - 多次失败后提示用户检查网络
