# 后端开发环境配置

## 1. 必需软件

### Python 3.12
- 推荐使用 Anaconda/Miniconda
- 下载：https://www.anaconda.com/download

### Redis（用于 Celery 任务队列）
- **Windows 推荐方式：WSL + Redis**
- 详见下方"Redis 安装"部分

## 2. 创建 Python 环境

```bash
conda create -n structural-design python=3.12
conda activate structural-design
```

## 3. 安装后端依赖

```bash
cd structural-design-system/backend
pip install -r requirements-api.txt
```

### 关键依赖版本
- FastAPI==0.109.0
- SQLAlchemy==2.0.25
- Celery==5.3.6
- Redis==5.0.1
- bcrypt==4.0.1（重要：必须是 4.0.1）
- python-multipart==0.0.9
- ezdxf==1.1.3

## 4. 数据库配置

### 开发环境（SQLite）
在项目根目录创建 `.env` 文件：

```bash
DATABASE_URL=sqlite:///./structural_design.db
REDIS_URL=redis://localhost:6379/0
SECRET_KEY=dev-secret-key-change-in-production
OUTPUT_DIR=./output
```

### 生产环境（PostgreSQL）
```bash
DATABASE_URL=postgresql://user:password@localhost/structural_design
REDIS_URL=redis://localhost:6379/0
SECRET_KEY=your-secret-key-here
OUTPUT_DIR=/path/to/output
```

## 5. Redis 安装

### Windows（推荐 WSL）

1. 安装 WSL（PowerShell 管理员）：
```bash
wsl --install
```

2. 重启电脑后，打开 Ubuntu

3. 在 Ubuntu 中安装 Redis：
```bash
sudo apt update
sudo apt install redis-server
```

4. 启动 Redis：
```bash
redis-server
```

### macOS
```bash
brew install redis
brew services start redis
```

### Linux
```bash
sudo apt install redis-server
sudo systemctl start redis
```

## 6. 启动后端服务

### 启动 API 服务器
```bash
cd structural-design-system
uvicorn backend.api.main:app --reload --host 0.0.0.0 --port 8000
```

### 启动 Celery Worker（新终端）
```bash
cd structural-design-system
celery -A backend.tasks.celery_app worker --loglevel=info --pool=solo
```

**注意：**
- Windows 必须使用 `--pool=solo`
- 需要同时运行 Redis、API 服务器和 Celery Worker

## 7. 验证安装

访问 http://localhost:8000/health 应返回：
```json
{"status": "healthy"}
```

访问 http://localhost:8000/docs 查看 API 文档

## 8. 常见问题

### bcrypt 版本错误
如果遇到 `AttributeError: module 'bcrypt' has no attribute '__about__'`：
```bash
pip install bcrypt==4.0.1
```

### Redis 连接失败
确保 Redis 服务正在运行：
```bash
redis-cli ping  # 应返回 PONG
```

### Celery 无法启动
Windows 用户必须使用 `--pool=solo` 参数
